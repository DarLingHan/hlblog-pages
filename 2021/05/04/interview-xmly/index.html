for-of、foreach、for in、map的区别

tips: 检查对象本身的属性hasOwnProperty()或者getOwnPropertyNames()或者propertyIsEnumerable

  • forEach——>>
  1. 只能遍历数组;
  2. 对数组的每个元素都执行指定的函数;
  3. 返回值undefined;
  4. 不能使用break跳出循环,使用return跳出本次循环，不会跳出循环;
  • for in——>>
  1. 以任意顺序遍历一个对象的属性(除Symbol以外);
  2. 可以使用break跳出循环;
  3. 代码中使用return会报错(VM852:4 Uncaught SyntaxError: Illegal return statement);
  4. index索引为字符串数字类型;遍历数组所有的可枚举属性,包括原型上的。
  • for of——>>
  1. 仅支持可迭代对象(Array、Map、Set、String、TypedArray、arguments对象等等也就是类数组对象也就是具有iterator接口);
  2. 不能使用return(VM852:4 Uncaught SyntaxError: Illegal return statement)
  • map——>>
  1. 只能遍历数组;返回创建的新数组,不改变元素组

什么是闭包，构成闭包依赖了哪些语言特性?

  • 就是函数外部能够读取函数内部的变量，那这个函数就是闭包；依赖了js的局部变量和全部变量的特性，函数里可以访问全局变量，但是不能外部不能访问局部变量。
  • 闭包的用途：
  1. 可以读取函数内部的变量；让这些变量的值始终保持在内存中。

new关键字做了什么

  • 创建一个新对象
  • 将构造函数的原型对象prototype指向给这个对象的隐式proto
  • 将构造函数的属性、方法复制给这个对象
  • 返回这个对象（如果没有返回值的话）

tips: 更新一个小问题当构造函数中的this碰到return时,如果返回值是一个对象，那么this指向的就是那个返回的对象，如果返回值不是一个对象那么this还是指向函数的实例。
还有一点就是虽然null也是对象，但是在这里this还是指向那个函数的实例，因为null比较特殊。

原型链

访问所以构造函数的属性和方法
一条实列与原型对象之间的联调
创建一个构造函数就会有一个protoType指向它的原型对象，而原型对象又有一个constructor指向它的构造函数。构造函数的实例有一个隐式_proto_指向构造函数的原型对象。
原型对象的属性、方法被所有实例所公用；构造函数中的属性、方法是实例的属性和方法，只出现在实例中。

举例——构造函数3想同时拥有构造函数1，2的属性和方法，那么就将构造函数1的实例指向构造函数2的原型对象，将构造函数2的实例指向构造函数3的原型对象，那么就可以在构造函数3的实例中访问构造函数1、构造函数2的属性方法了。

this的问题

  • this的指向在函数定义的时候是确定不了的，只有函数执行的时候才能确定this到底指向谁，实际上this的最终指向的是那个调用它的对象

简述call/apply的区别

  • apply (新this对象,参数数组)
  • call (新this对象,参数,参数,参数…)
  • bind 则是返回改变了this指向后的函数，不执行该函数。

setTimeout和setInterval做轮询的区别

  • setTimeout 延时器
  • setInterval 定时器

手写实现防抖和截流

  • 防抖 一定时间内只会触发一次，多次操作，会重新计时，直到最后只执行一次。
  • 节流 以一定频率执行事件，至少执行一次。

柯里化

  • 只传递给函数一个参数调用它，返回一个函数处理剩下的参数
  • 把接受多个参数的函数转换成接受单一参数的函数，并且返回一个函数，这个函数接受余下的参数和返回结果。
  • javascript闭包的特性，是我们将函数柯里化的基础
  • 柯里化具有延迟计算、参数复用、动态生成函数的作用
